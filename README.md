# 机器人上下位机传输协议：

```
SOF1 | SOF2 | CFG | SEQ | SRC | DST | FUNC | CMD | LEN(2B) | PAYLOAD | CRC16(2B)
```

| 字段    | 长度 | 说明                            |
| ------- | ---- | ------------------------------- |
| SOF1    | 1B   | 帧起始字节1，固定 `0xA5`      |
| SOF2    | 1B   | 帧起始字节2，固定 `0x5A`      |
| CFG     | 1B   | 配置位                          |
| SEQ     | 1B   | 序号 0~255                      |
| SRC     | 1B   | 源地址（上位机默认 `0x01`）   |
| DST     | 1B   | 目的地址（下位机默认 `0x10`） |
| FUNC    | 1B   | 功能域                          |
| CMD     | 1B   | 子命令                          |
| LEN     | 2B   | PAYLOAD 长度（小端）            |
| PAYLOAD | 可变 | 数据区                          |
| CRC16   | 2B   | 校验值（小端）                  |

1.sof1，sof2，用于识别一帧起始

2.cfg字段，包含协议版本，后续升级后用于对齐协议版本

包含ack信息，决定该信息是否需要对方确认收到，紧急刹车时，设置need_ack==1,即需要应答

3.seq字段，用于标记数据帧序号，1-255循环，用于接收方如果需要作出应答，标定序号方便让应答对应

4.src，dst字段，标记是哪一部分发起数据请求，需要哪一部分接收，便于在同一串口总线上分别控制各个部件的单片机，类似can通讯，后续挂载塔吊，钩爪，取料盘时，便于控制

5.func字段，标记此数据帧内的命令作用于哪一部分，0x01为下位机对上位机，用于主动传输imu传感器数据

    0x02上位机对底盘发出命令操控，0x03上位机对塔吊，钩爪发出命令操控（还未定义命令具体内容，暂时不可调用），0x04用于上位机主动请求imu数据（还未定义命令具体内容，暂时不可调用）

6.cmd字段，包含具体要发布什么类型的命令，目前func0x02 下，有设置速度，设置位置，获取状态

    设置速度，收到命令后，分割出payload区数据，传入handle_chassis_set_vel函数，将小段字节数据转换为包含四个轮子的角速度的int16数组

    单位换算，const float scaled = (float)raw[i] * CHASSIS_CMD_LSB_RAD_PER_SEC，将速度乘一个换算系数CHASSIS_CMD_LSB_RAD_PER_SEC，具体单位制，可以自己改，本质

    就是个浮点数，默认是0.01rad/s ，改的话，需要将下面的速度限制函数一块改了

    速度限制，clamp_speed(scaled)将角速度传入这个函数，默认角速度高于200rad/s，判断该数据帧数据无效，抛弃该数据帧 200rad/s也是可以改的，修改

    CHASSIS_MAX_RAD_PER_SEC

    解析完毕后，使用Chassis_GetSetpoint(ChassisSetpoint *out)函数，获取解析后指令内容，返回一个布尔值，表示该组数值是否有效

    主函数中，将获取到的角速度，换算成rpm转速，正转速给ccw布尔值赋值为false，代表顺时针，反之逆时针

    附带最终计算完毕的完整给电机的速度指令格式：payload[0] = 方向 (0x01: CCW, 0x00: CW)
										  payload[1-2] = 速度 RPM (大端序)
										  payload[3] = 加速度档位 (0=直接启动)
										  payload[4] = 同步标志 (0x01: 启用)

    四电机同步启动：Emm42_SyncStart函数，传入一个指向电机句柄的指针，可以直接传入g_emm42_motors[0]，因为底盘四个电机全部共用uart1串口，发送广播命令就可以一次性4个

    电机发送命令，想给某一个电机，只要在函数参数中，设置sync参数为false，然后给某序号电机制定ccw，rpm等参数就好

    设置位置指令，数据处理跟设置速度相同，只是使用pwm来控制电机转多少小下，默认16细分

    imu信息，从现有的JY901B.c里的g_jy901_raw变量读取原始的32字节数据 包含：

7.payload字段，cmd部分已经说明了

8.crc16校验，发送方先把前面的指令内容算一遍，接收方收到数据后，自己再算一遍，如果两边不一致，就是数据错误，丢弃该数据帧

不用can的原因：imu数据有32字节，can2b一帧只有8字节，得拆成多个包发，太麻烦，丢包全部都得重来，不如直接串口，自己搞协议

为何不采集编码器数据：步进电机，转速下多少就是多少，打滑了还是记原来的圈数，直接用上位机传的数据就好，反而更快
