# 串口协议规范（下位机）

> 依据 `serial_protocol.h / serial_protocol.c` 实现整理。  
> 所有多字节字段为**小端序**。  
> 不符合本规范的帧，下位机将**直接丢弃**。

---

## 1. 帧结构（标准格式）

```
SOF1 | SOF2 | CFG | SEQ | SRC | DST | FUNC | CMD | LEN(2B) | PAYLOAD | CRC16(2B)
```

| 字段 | 长度 | 说明 |
|------|------|------|
| SOF1 | 1B | 帧起始字节1，固定 `0xA5` |
| SOF2 | 1B | 帧起始字节2，固定 `0x5A` |
| CFG  | 1B | 配置位（见 2.1） |
| SEQ  | 1B | 序号 0~255 |
| SRC  | 1B | 源地址（上位机默认 `0x01`） |
| DST  | 1B | 目的地址（下位机默认 `0x10`） |
| FUNC | 1B | 功能域（见 3.1） |
| CMD  | 1B | 子命令（见 3.2） |
| LEN  | 2B | PAYLOAD 长度（小端） |
| PAYLOAD | 可变 | 数据区 |
| CRC16 | 2B | 校验值（小端） |

---

## 2. CFG 配置位

### 2.1 位定义

```
bit7..bit4 : 协议版本（PROTOCOL_VERSION）
bit3..bit2 : 预留，必须为 0
bit1       : IS_ACK（1=本帧为ACK）
bit0       : NEED_ACK（1=请求对方ACK）
```

### 2.2 合规取值示例

| 场景 | CFG |
|------|-----|
| 普通帧（不需ACK） | `0x10` |
| 普通帧（需ACK） | `0x11` |
| ACK帧 | `0x12` |

---

## 3. 功能与命令

### 3.1 功能域（FUNC）

| FUNC | 含义 |
|------|------|
| `0x01` | 系统 |
| `0x02` | 底盘 |
| `0x03` | 舵机/机械臂 |
| `0x04` | 传感器 |

### 3.2 底盘命令（FUNC=0x02）

| CMD | 名称 | 方向 |
|-----|------|------|
| `0x01` | 设置速度（SET_VEL） | 上位机 → 下位机 |
| `0x02` | 设置位置（SET_POS） | 上位机 → 下位机 |
| `0x03` | 获取状态（GET_STATE） | 上位机 → 下位机 |

### 3.3 传感器命令（FUNC=0x04）

| CMD | 名称 |
|-----|------|
| `0x01` | 获取 IMU（GET_IMU） |

> 未定义的 CMD（例如 `0x03`）**不得使用**。

---

## 4. PAYLOAD 结构

### 4.1 SET_VEL（FUNC=0x02, CMD=0x01）

| 字段 | 类型 | 数量 | 说明 |
|------|------|------|------|
| v1 | int16 | 1 | 轮1速度 |
| v2 | int16 | 1 | 轮2速度 |
| v3 | int16 | 1 | 轮3速度 |
| v4 | int16 | 1 | 轮4速度 |

- **总长度**：8B  
- **小端序**  
- 正负号表示方向

---

### 4.2 SET_POS（FUNC=0x02, CMD=0x02）

| 字段 | 类型 | 数量 | 说明 |
|------|------|------|------|
| p1 | int32 | 1 | 轮1目标脉冲 |
| p2 | int32 | 1 | 轮2目标脉冲 |
| p3 | int32 | 1 | 轮3目标脉冲 |
| p4 | int32 | 1 | 轮4目标脉冲 |

- **总长度**：16B  
- **小端序**

---

### 4.3 GET_STATE 响应（FUNC=0x02, CMD=0x03）

| 字段 | 类型 | 数量 | 说明 |
|------|------|------|------|
| v1..v4 | int16 | 4 | 轮速状态（来自最近一次速度命令） |
| p1..p4 | int32 | 4 | 轮位移/脉冲（来自最近一次位置命令） |

- **总长度**：24B  
- **小端序**  

> 说明：此处返回的是**缓存指令状态**，不是驱动器实际反馈。  
> 若需真实编码器/驱动反馈，请扩展驱动回读命令。

## 5. CRC16（作用与计算方式）

### 5.1 作用（强制）
CRC16 用于检测**帧传输误码**。  
**校验失败的帧必须被丢弃，不得执行命令。**

### 5.2 计算范围（严格）
CRC16 覆盖范围为：

```
SOF1 .. PAYLOAD（含头部与数据，不含 CRC 字段）
```

### 5.3 算法（与固件实现一致）
**CRC-16/IBM（又称 Modbus 变体）**

| 参数 | 取值 |
|------|------|
| 多项式 Poly | `0x8005`（反射实现等价 0xA001） |
| 初值 Init | `0xFFFF` |
| RefIn | true |
| RefOut | true |
| XorOut | `0x0000` |

固件实现（节选）：
```c
uint16_t crc = 0xFFFF;
crc ^= buf[i];
for (bit=0..7):
    if (crc & 0x0001) crc = (crc >> 1) ^ 0xA001;
    else crc >>= 1;
```

## 6. ACK 帧定义

### 6.1 触发条件
当请求帧 `CFG.bit0 = NEED_ACK` 时，下位机**必须返回 ACK**。

### 6.2 ACK 帧结构
ACK 帧与标准帧结构一致，且必须满足：

| 字段 | ACK 要求 |
|------|----------|
| CFG | `IS_ACK=1`, `NEED_ACK=0` |
| SEQ | **与请求帧一致** |
| SRC | 下位机地址 |
| DST | 请求帧 SRC |
| FUNC | 与请求帧一致 |
| CMD | 与请求帧一致 |
| LEN | 固定 `1` |
| PAYLOAD | 1 字节结果码 |

### 6.3 ACK 结果码（Payload）

| 值 | 含义 |
|----|------|
| `0x00` | OK |
| `0x01` | ERR_UNSUPPORTED |
| `0x02` | ERR_PARAM |
| `0x03` | ERR_INTERNAL |

---

## 7. 打包流程（严格）

1. 设置 `SOF1=0xA5, SOF2=0x5A`  
2. 设置 `CFG`（版本号 + ACK 标志）  
3. 填写 `SEQ/SRC/DST/FUNC/CMD`  
4. 计算 `LEN`（PAYLOAD 字节数，小端）  
5. 拼接 `Header + Payload`  
6. 计算 CRC16 并按小端追加

假设：
- v1=v2=v3=v4=100  
- SEQ=0x01  
- CFG=0x10  
- SRC=0x01，DST=0x10  

```
A5 5A 10 01 01 10 02 01 08 00
64 00 64 00 64 00 64 00
CRC_L CRC_H
```
